Diseño de la App: Registro de Datos

Registro ProductoDBD ID_Articulo: Cadena // CLAVE PRINCIPAL Nombre_Articulo: Cadena Tipo_Categoria: Cadena Es_Licencia: Lógico Precio_Celulas: Entero Precio_Fragmentos: Entero Fin Registro

Justificación de la Clave: Se eligió ID_Articulo como clave principal porque en un e-commerce es indispensable contar con un identificador único, irrepetible y alfanumérico para cada producto. Esto evita confusiones en la base de datos (por ejemplo, diferenciar entre el personaje "Leon S. Kennedy" y un cosmético o suéter que lleve el mismo nombre) garantizando que las transacciones sean exactas.
